## 1. 产品概述 (Executive Summary)

### 产品定位
核心网黄金指标分析系统是面向电信运营商核心网运维团队的智能监控平台，基于1分钟粒度的网元黄金指标，提供告警规则配置、指标报表可视化、告警事件监控三合一能力，帮助运维人员从"被动等投诉"转变为"主动秒级发现"核心网故障。

### 目标用户
| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 运维值班工程师 | 7×24值班，需快速发现定位故障 | 秒级发现异常，一眼定位故障网元 |
| 网元维护管理人员 | 按省份/专业分工，需统计分析 | 按维度查询报表，导出汇报 |
| 运维主管/领导 | 需全局态势感知和决策支持 | 一屏掌握全局健康状态 |

### 核心价值
- **1分钟粒度**：比传统5/15分钟采集更早发现故障苗头
- **智能规则引擎**：同比/环比/多指标联合判决，降低误报漏报
- **可视化定位**：告警趋势叠加指标折线图，一眼识别异常

### 成功指标 (SMART Metrics)
| 指标 | 基线 | 目标 | 测量方式 |
|------|------|------|----------|
| 故障发现时效 | 当前15-30分钟 | < 3分钟（从指标异常到告警触发） | 告警触发时间 - 指标异常时间 |
| 告警误报率 | 当前约30% | < 10% | 误报告警数 / 总告警数 |
| 规则配置时间 | 无（手工盯屏） | < 5分钟（使用模板） | 从创建规则到规则生效的时长 |
| 报表查询响应 | — | P95 < 3秒 | APM监控 |
| 系统日活率 | — | > 80%值班人员日活 | 日活用户数 / 值班人员总数 |

## 2. 问题与目标 (Problem Statement)

### 问题定义
核心网网元（AMF/SMF/UPF/MME等）拥有最关键的"黄金指标"（如注册成功率、会话建立率、切换成功率等），采集粒度为1分钟。当前运维面临：

1. **发现慢**：依赖5/15分钟采集或人工巡检，故障发现延迟15-30分钟
2. **误报多**：缺乏滑动窗口和联合判决机制，链路闪断导致大量无效告警
3. **定位难**：告警与指标趋势分离，需在多个系统间切换才能确认故障根因
4. **配置门槛高**：阈值规则配置复杂，新手无法独立完成

### 用户影响 (User Impact)
- 运维人员需7×24盯屏，工作强度大且容易遗漏
- 故障发现延迟导致客户投诉和SLA违约
- 告警风暴时无法快速定位根因，MTTR（平均修复时间）拉长

### 业务影响 (Business Impact)
- 核心网故障每分钟影响数万用户，30分钟延迟 = 数十万用户受影响
- SLA违约可能导致运营商面临千万级罚款
- 运维人力成本高，告警风暴时需临时加派人手

### 产品目标
1. **快速发现**：1分钟粒度指标 + 秒级告警触发，3分钟内发现异常
2. **精准告警**：滑动窗口 + 多指标联合判决，误报率降至10%以下
3. **直观定位**：告警趋势叠加指标折线图，一眼识别故障网元
4. **零成本上手**：预置规则模板，5分钟内完成首条规则配置

## 3. 用户角色与权限

### 角色定义
| 角色 | 角色标识 | 数据范围 | 说明 |
|------|----------|----------|------|
| 集团运维 | GROUP_ADMIN | 全国所有网元 | 可查看全国数据，可管理所有规则 |
| 省级运维 | PROVINCE_ADMIN | 本省网元 | 只能查看本省数据，可管理本省规则 |
| 省级值班 | PROVINCE_OPERATOR | 本省网元 | 只能查看本省数据，可确认/处理告警 |
| 只读用户 | VIEWER | 按省份授权 | 只能查看报表和监控，不能操作 |

### 权限矩阵
| 功能 | 集团运维 | 省级运维 | 省级值班 | 只读用户 |
|------|----------|----------|----------|----------|

| 查看本省数据 | ✅ | ✅ | ✅ | ✅ |
| 创建/编辑/删除规则 | ✅ | ✅（本省） | ❌ | ❌ |
| 查看规则 | ✅ | ✅ | ✅ | ✅ |
| 复制规则 | ✅ | ✅（本省） | ❌ | ❌ |
| 启用/停用规则 | ✅ | ✅（本省） | ❌ | ❌ |
| 确认/处理告警 | ✅ | ✅ | ✅ | ❌ |
| 导出报表 | ✅ | ✅ | ✅ | ✅ |
| 管理用户 | ✅ | ❌ | ❌ | ❌ |

### 数据隔离规则
- 用户登录后，后端根据角色自动注入省份过滤条件
- 集团用户：不注入过滤，可查全国
- 省级用户：自动注入 `province_code = 用户所属省份`
- 前端筛选器中，省级用户不显示省份选择器（自动锁定）
- API层面强制校验数据范围，前端动态渲染仅为体验优化

## 4. 功能模块 (Functional Requirements)

### 模块一：黄金指标规则配置（页面一）

#### REQ-001: 规则列表与基础管理
- **场景**：运维人员需要查看、创建、编辑、删除告警规则
- **描述**：
  - 按网元类型分Tab展示规则列表
  - 列表字段：规则ID、规则名称、厂家（华为/中兴/爱立信，单选）、网元类型、应用场景、条件（运算符使用＜≤≥＞符号）、等级、状态（启用/停用）、创建人、创建时间、最新修改时间、操作
  - 操作列包含：查看、编辑、删除、复制（一键生成草稿规则）
  - 含故障派单标签的规则不允许编辑和删除操作
  - 条件表达式中运算符使用关系符号（＜≤≥＞），不使用lt/gt缩写
  - 报表不展示「触发窗口」字段
  - 支持按规则名称搜索、按状态筛选、按厂家筛选、按应用场景筛选、按创建人（Falcomm账号）筛选
  - 支持批量启用/停用规则
  - 查看规则时支持翻页浏览（上一条/下一条切换）
- **优先级**：Must have (P0)
- **依赖**：无
- **验收标准**：
  1. 规则列表加载时间 P95 < 1秒
  2. 支持至少500条规则的列表展示与分页
  3. 批量操作最多支持50条规则
  4. 含故障派单标签的规则，编辑和删除按钮置灰且不可点击
  5. 复制规则后自动生成草稿状态的新规则
  6. 查看模式支持翻页，序号显示正确
  7. 厂家筛选下拉选项：华为/中兴/爱立信，支持清空
  8. 应用场景筛选下拉选项：故障派单/业务受损监控/动网自动化，支持清空
  9. 创建人筛选支持输入Falcomm账号模糊搜索
- **预估工作量**：~10h

#### REQ-002: 规则模板市场
- **场景**：新用户首次使用，需要快速配置告警规则
- **描述**：
  - 预置主流网元类型的黄金指标告警规则模板（覆盖20种网元类型×10个指标的核心场景）
  - 用户可从模板一键创建规则，微调参数后保存
  - 模板按网元类型分组展示，支持搜索
  - 模板字段：模板名称、网元类型、指标、条件类型、默认阈值、说明
- **优先级**：Must have (P0)
- **依赖**：REQ-001
- **验收标准**：
  1. 预置模板覆盖20种网元类型，每种至少1个模板
  2. 从模板创建规则到规则生效 < 5分钟
  3. 模板创建的规则可正常触发告警
- **预估工作量**：~6h

#### REQ-003: 单指标阈值条件配置
- **场景**：运维人员需要为单个指标设置告警阈值
- **描述**：
  - 支持阈值类型：绝对值阈值、环比（较上个周期）、同比（较前一天同期）
  - 环比选项增加备注提醒"较上个周期"，同比选项增加备注提醒"较前一天同期"
  - 同比/环比支持设置偏移百分比（如"环比下降超过20%"）
  - 阈值支持配置上下限（如"注册成功率 < 95% 或 > 100%"）
  - 实时显示指标含义说明和单位
  - 运算符选项：`<` `<=` `>` `>=`（去掉 `=` 和 `!=`）
  - 单位跟随所选指标，不可手动选择，展示为只读状态并标注"跟随指标"
- **优先级**：Must have (P0)
- **依赖**：REQ-001
- **验收标准**：
  1. 三种阈值类型均可正常配置并保存
  2. 同比/环比计算逻辑正确（与后端一致）
  3. 阈值配置界面有即时校验（如环比偏移百分比范围0-100%）
  4. 运算符下拉框仅展示 `<` `<=` `>` `>=` 四个选项
  5. 单位自动跟随指标，显示为只读，标注"跟随指标"
  6. 环比选项 hover 显示"较上个周期"，同比选项 hover 显示"较前一天同期"
- **预估工作量**：~8h

#### REQ-004: 滑动时间观察窗口
- **场景**：防止链路闪断等瞬态异常导致误发告警
- **描述**：
  - 配置连续N个采集点（1-10个，即1-10分钟）满足条件才触发告警
  - 支持配置恢复观察窗口（连续N个点恢复才解除告警）
  - 规则预览区展示窗口效果说明文字
- **优先级**：Must have (P0)
- **依赖**：REQ-003
- **验收标准**：
  1. 观察窗口参数可正确配置并保存
  2. 告警触发逻辑严格遵循窗口设置（不满足窗口数不触发）
  3. 恢复窗口逻辑正确
- **预估工作量**：~6h

#### REQ-005: 多指标联合判决
- **场景**：需要多个指标同时异常才触发告警，提高告警精准度
- **描述**：
  - 支持添加多个指标条件，通过 AND/OR 逻辑组合
  - 每个指标条件独立配置阈值类型和阈值
  - 可视化逻辑图展示条件关系（AND门/OR门）
  - 最多支持5个指标条件联合
- **优先级**：Should have (P1)
- **依赖**：REQ-003
- **验收标准**：
  1. AND/OR逻辑可正确配置
  2. 多条件联合判定逻辑与后端一致
  3. 逻辑图实时更新反映条件变化
- **预估工作量**：~10h

#### REQ-006: 规则生效时间段设置
- **场景**：部分规则仅在工作时间或特定时段生效
- **描述**：
  - 支持设置每日生效时间段（如08:00-20:00）
  - 支持设置每周生效日（如周一至周五）
  - 支持设置例外日期（如节假日不生效）
  - 日历视图展示规则生效时间
- **优先级**：Should have (P1)
- **依赖**：REQ-001
- **验收标准**：
  1. 时间段配置可正确保存
  2. 非生效时段不触发告警
  3. 日历视图正确反映生效时间
- **预估工作量**：~6h

#### REQ-007: 规则历史效果模拟
- **场景**：配置规则后想预览该规则在过去时段的命中情况
- **描述**：
  - 配置规则后，点击"效果模拟"，选择过去24h/7天
  - 展示该规则在选定时段的命中次数、命中时段
  - 与实际告警对比，显示新增命中和已命中
- **优先级**：Could have (P2)
- **依赖**：REQ-003, REQ-004
- **验收标准**：
  1. 模拟结果与实际回放数据一致
  2. 模拟查询响应时间 < 10秒
- **预估工作量**：~8h

### 模块二：黄金指标报表（页面二）

  3. 拖拽缩放、hover交互正常
  4. 数值型和百分比型指标不在同一Y轴
- **预估工作量**：~12h

  - 支持按时间倒序排列（最新在前）
  - 支持按告警等级筛选和排序
- **优先级**：Must have (P0)
- **依赖**：无
- **验收标准**：
  1. 筛选条件联动正确
  2. 列表加载时间 P95 < 2秒
  3. 告警等级颜色与定义一致
- **预估工作量**：~8h

#### REQ-020: 大区级衍生告警（告警层聚合）
- **场景**：同一大区内多个省份出现同类黄金指标告警时，自动生成大区级衍生告警，帮助运维人员快速识别区域性故障
- **描述**：
  - **聚合维度**：大区 + 网元类型 + 黄金指标（同一大区、同一网元类型、同一黄金指标）
  - **触发条件**：在指定时间窗口内，某大区有 ≥2 个省份产生同类告警（同一网元类型 + 同一黄金指标），则生成一条大区级衍生告警
  - **全局阈值配置**：触发阈值（省份数量，默认2）和时间窗口（默认30分钟）存储在 `system_config` 表中，全局统一，不需要用户配置 UI
  - **告警等级**：取子告警中的最高等级，自动升级（子告警等级变化时实时更新）
  - **子告警追加**：当衍生告警未消除期间，同一大区又有新的相关网元级告警产生时，追加为子告警到已有衍生告警，而不产生新的衍生告警（避免告警瀑布）
  - **清除逻辑**：仅当所有子告警都被清除后，衍生告警才自动清除
  - **操作限制**：衍生告警不需要确认/转派/升级等人工操作，删除相关操作按钮
  - **6大区**：华南/华北/华东/西南/东北/华中，Pool ⊆ Province ⊆ Region
- **优先级**：Must have (P0)
- **依赖**：REQ-014, REQ-016
- **验收标准**：
  1. ≥2省份同类告警在时间窗口内正确触发大区级衍生告警
  2. 衍生告警等级 = max(子告警等级)，实时更新
  3. 新子告警追加到已有衍生告警，不产生重复衍生告警
  4. 所有子告警清除后衍生告警自动清除
  5. 衍生告警无确认/转派/升级操作按钮
  6. 全局阈值从 system_config 正确读取
- **预估工作量**：~16h

#### REQ-021: Pool级衍生告警（指标层聚合）
- **场景**：需要对Pool内多个网元的指标值进行聚合计算后与基线比较，发现Pool级别的指标异常
- **描述**：
  - **聚合维度**：Pool + 网元类型 + 黄金指标
  - **聚合方式**：指标层聚合（先聚合指标值，再比较阈值），区别于大区级的告警层聚合（先有告警，再计数）
  - **每条规则独立配置**：
    - 聚合函数：sum（求和）/ avg（平均）/ max（最大）/ min（最小）
    - 源指标：选择Pool内网元的黄金指标
    - 比较基线：昨日同期（yesterday_same_period）/ 前一正常周期（previous_normal_cycle）/ 绝对阈值（absolute）
    - 比较运算符：< / <= / > / >= / = / !=
    - 阈值：与基线比较的偏移量或绝对值
    - 观察窗口：连续N个采集点满足条件才触发
    - 告警等级：1-4级
  - **示例规则**：统计AMF POOL内各网元的"AMF注册态用户数"之和，连续3个周期与昨日同期比较，下降超过10%则触发
  - **子告警追加**：与大区级一致，衍生告警未消除期间新产生的网元级告警追加到已有衍生告警
  - **清除逻辑**：仅当所有子告警都被清除后，衍生告警才自动清除
  - **告警等级**：取子告警中的最高等级，自动升级
  - **操作限制**：衍生告警不需要确认/转派/升级等人工操作
  - **同一原始告警可匹配多条聚合规则**：所有匹配的规则均生成衍生告警，不互斥
- **优先级**：Must have (P0)
- **依赖**：REQ-014, REQ-020
- **验收标准**：
  1. Pool级聚合规则可独立配置聚合函数/基线/阈值/窗口
  2. sum/avg/max/min 四种聚合函数计算结果正确
  3. 昨日同期/前一正常周期/绝对阈值三种基线比较逻辑正确
  4. 子告警追加和清除逻辑与大区级一致
  5. 同一原始告警匹配多条规则时均生成衍生告警
  6. 衍生告警无确认/转派/升级操作按钮
- **预估工作量**：~20h

### 实体关系
```
alarm_rule 1──N alarm_event（一条规则可触发多条告警事件）
ne_info 1──N golden_metric（一个网元有多条指标时序数据）
ne_info 1──N alarm_event（一个网元可有多条告警事件）
alarm_rule N──1 ne_type（规则按网元类型分组）
aggregation_rule 1──N derived_alarm（一条聚合规则可生成多条衍生告警）
derived_alarm 1──N derived_alarm_child（一条衍生告警包含多个子告警）
derived_alarm_child N──1 alarm_event（子告警映射到原始告警事件）
ne_info N──1 region（网元归属于大区，通过 province_code 关联）
```

## 6. 状态机

### 告警事件状态流转
```
pending ──确认──→ confirmed ──开始处理──→ processing ──恢复──→ resolved
  │                  │                     │
  │                  │                     └──关闭──→ closed
  │                  │
  └──标记误报──→ false_alarm
  │
  └──转派──→ pending（处理人变更）
```

**流转规则**：
- pending → confirmed：运维人员确认告警，需填写处理说明
- confirmed → processing：开始处理告警
- processing → resolved：指标恢复正常，系统自动或人工标记恢复
- processing → closed：人工关闭告警
- pending → false_alarm：标记为误报
- 任何状态 → pending：转派给其他人员（重置为待处理）

## 7. 用户流程

### 主流程：从发现告警到定位处理
```
1. 运维人员登录系统
2. 进入监控报表页，查看当前告警摘要
3. 发现1级告警事件（AMF注册成功率异常）
4. 点击告警事件 → 展开详情面板
5. 查看告警指标趋势图 → 跳转报表页该网元该时段
6. 报表页查看折线图，确认指标下降趋势和阈值对比
7. 查看关联告警（同Pool其他AMF是否异常）
8. 点击异常点 → 跳转规则页查看规则配置
9. 确认规则合理后返回监控页
10. 确认告警，填写处理说明，开始处理
11. 处理完成后标记恢复/关闭
```

### 分支流程：规则配置
```
1. 运维人员进入规则配置页
2. 选择网元类型 → 查看已有规则/模板
3a. 从模板创建 → 选择模板 → 微调参数 → 保存
3b. 自定义创建 → 选择指标 → 配置阈值 → 配置窗口 → 配置联合条件 → 保存
4. 效果模拟（可选）→ 查看历史命中情况
5. 启用规则
```

### 分支流程：报表导出
```
1. 运维人员进入报表页
2. 设置筛选条件（省份/类型/时间）
3. 查看折线图/表格
4. 点击导出
5a. 小数据量 → 直接下载Excel
5b. 大数据量 → 创建异步任务 → 等待通知 → 下载
```

### 异常流程
| 场景 | 处理方式 |
|------|----------|
| 数据源中断 | 页面显示"数据延迟"提示，展示最后可用数据时间戳 |
| 告警引擎异常 | 监控页显示"告警服务异常"横幅，不隐藏已有告警 |
| 导出任务超时 | 通知用户导出失败，提供重试链接 |
| 权限变更 | 操作时校验权限，无权限时提示并隐藏操作按钮 |

| 异常码 | 说明 | 处理方式 |
