## 一、全局设计规范

### 1.1 布局框架

**顶部导航 + 标签栏 + 内容区**，不使用左侧导航栏：

```
+-------------------------------------------------+
|  LOGO  黄金指标分析系统  [规则配置] [指标报表] [监控报表]  🔔 告警(3)  张三 ▾  |
+-------------------------------------------------+
|  [规则配置 ✕]  [指标报表 ✕]  [监控报表]         |
+-------------------------------------------------+
|  Main Content (底色 #f1f3f6, padding: 12px)     |
|  +-------------------------------------------+ |
|  |  White Card Container (白色卡片, 圆角)    | |
|  |  - 无默认padding，由业务组件控制          | |
|  +-------------------------------------------+ |
+-------------------------------------------------+
```

- **顶部菜单栏**：高度50px，padding 2px，固定顶部不滚动
  - 左侧：Logo + 系统名称
  - 中部：导航菜单项（规则配置、指标报表、监控报表），水平排列，间距2px，内边距 0 20px
  - 右侧：告警铃铛（显示当前待处理告警数）、用户头像/退出登录/系统设置
- **标签栏**：高度36px，位于菜单栏下方，支持多标签页切换、关闭、右键菜单
- **内容区**：底色 `#f1f3f6`，内边距 12px，白色卡片铺满
- **不使用**：左侧导航栏、面包屑导航、标题下文字描述

### 1.5 组件通用样式

#### 顶部菜单栏

- 位置：固定顶部，高度50px，padding 2px
- 菜单项：水平排列，间距2px，内边距 0 20px
- 选中态：背景 `#d9e9fb`，文字 `#006be6`，底部无边框或下划线
- 悬浮态：背景 `#f4f4f5`
- 正常态：背景透明，文字 `#1f2937`

#### 标签栏

- 位置：内容区顶部，菜单栏下方
- 选中态：背景 `#d9e9fb`，文字 `#006be6`，上左和上右5px圆角
- 正常态：背景透明，文字 `#1f2937`
- 悬浮态：背景 `#f4f4f5`
- 支持操作：关闭、刷新、右键菜单

#### 按钮

| 类型 | 背景 | 文字 | 边框 | 圆角 |
|------|------|------|------|------|
| 主按钮 | `#006be6` | 白色 | 无 | 6px |
| 默认按钮 | 透明 | `#1f2937` | `#d9d9d9` | 6px |
| 文本按钮 | 透明 | 主色 | 无 | — |
| 危险按钮 | `#ff4d4f` | 白色 | 无 | 6px |

- 按钮尺寸：middle 为默认，小布局使用 small
- 按钮文字必须有，图标作为辅助

#### 表单布局

- **单表单布局**：左侧 label（文字描述），右侧控件（select/input），中间冒号分隔，必填项 label 前红色 `*`
- **页面内表单**：Row/Col 栅格，一行最多 4 个表单项（每 Col 占 6 栅格）
- **弹窗内表单**：一行最多 2 个表单项（每 Col 占 12 栅格）
- **输入框**：圆角 6px，聚焦时边框主色 + 浅色阴影
- **查询/重置按钮**：在所有表单项最后展示，如当前行已有4项则换行靠右，否则占据一个表单位置随后显示，按钮 48px × 22px

#### 表格

- 默认样式：带边框（`bordered`）
- 列名（thead）：背景色 `#f1f5fa`，文字加粗（weight 600），文字颜色 `#1f2937`，所有列名不允许换行，需设置宽度
- 内容：所有文字居中显示
- 行悬浮：背景 `#fafafa`
- 操作列：固定在右侧
- 分页器：位于表格 **右下方**
- 空数据：显示 Ant Design 默认空状态

#### 弹窗（Modal）

- 圆角 12px，标题栏无底部边框
- 支持拖拽、全屏
- 遮罩：`rgba(0,0,0,0.45)`

#### 抽屉（Drawer）

- 从右侧滑出，宽度 480px（或自适应）
- 圆角，遮罩同弹窗

### 1.6 共享组件

#### 网元选择器（三页共用）

```
┌─────────────────────────────────────────────────────────────────┐
│ * 省份: [广东 ▾]   * 专业: [5G ▾]   * 类型: [AMF ▾]   * 厂商: [华为 ▾]  │
│   网元: [🔍 搜索网元名称/ID...              ]   [多选 ▾]        │
└─────────────────────────────────────────────────────────────────┘
```

- 逐级联动：选省份→过滤专业→过滤类型→过滤厂商
- 集团用户显示省份选择器，省级用户自动锁定（隐藏省份选择器）
- 网元支持模糊搜索+多选（下拉树形结构）
- 选择器状态跨页面同步
- 遵循表单布局规则：label 左侧，控件右侧

### 1.7 间距规范

- 基于 8px 网格，常用间距：4, 8, 12, 16, 20, 24, 32

### 1.8 设计红线（Do / Don't）

#### ✅ Do

- 所有文字尽量不换行，除非非常长的文字描述
- 同一水平线元素保持水平对齐，同一垂直线元素保持垂直对齐
- 顶部菜单选中态使用 `#d9e9fb` 背景 + `#006be6` 文字
- 表格列名背景 `#f1f5fa`，文字加粗，所有内容居中
- 分页器固定放置在表格右下方
- 页面内表单最多一行四列，弹窗内最多一行两列
- 单表单 label 左控件右，冒号分隔
- 内容区底色 `#f1f3f6` + 12px 内边距，白色卡片平铺满
- 按钮文字必须有，图标作为辅助
- 每个表单 label 和控件水平对齐
- 首先考虑局部滚动条，需要滚动的部分才滚动
- 表格操作按钮放在紧挨着表格的上方
- 实际内容区标题前合理使用图标
- 多使用圆角，卡片和组件不要太过生硬

#### ❌ Don't

- 不要出现左侧菜单栏（除非指定描述）
- 不要出现面包屑导航
- 不要出现一排超过四个表单项的布局
- 不要出现单个表单 label 和控件换行展示
- 不要出现实际内容区没有白色背景色的情况
- 不要出现白色卡片没有平铺满主要内容区的情况
- 标题下方不要出现文字描述

### 2.2 规则创建弹窗（自定义创建）

#### 2.2.0 步骤引导架构

采用分步式引导，将规则创建拆解为4个步骤：

```
┌─────────────────────────────────────────────────────────────┐
│  创建告警规则                                            ✕  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ① 基本信息 ──── ② 告警条件 ──── ③ 观察窗口 ──── ④ 生效设置 │
│     ●─────────────○──────────────○──────────────○           │
│                                                             │
│  ┌─────────────────────────────────────────────────────────┐│
│  │                                                         ││
│  │              （当前步骤内容区）                           ││
│  │                                                         ││
│  └─────────────────────────────────────────────────────────┘│
│                                                             │
│                              [取消]  [← 上一步]  [下一步 →] │
└─────────────────────────────────────────────────────────────┘
```

**步骤规则**：
- 步骤间可自由点击跳转（已完成步骤可回退修改）
- 每步必填项校验通过后才激活"下一步"按钮
- 最后一步按钮变为"效果模拟"+"保存"
- 编辑已有规则时，4步全部展开可自由切换
- 弹窗圆角12px，标题栏无底部边框
- 弹窗内表单：一行最多2个表单项（每Col占12栅格）

**条件编排交互**：

| 操作 | 交互方式 | 说明 |
|------|----------|------|
| 添加条件 | 点击「+ 添加条件」按钮 | 弹出指标选择下拉，选中后添加条件卡片 |
| 删除条件 | 点击卡片右上角 ✕ | 单条件不可删除（至少保留1个） |
| 调整逻辑关系 | 点击条件间的 AND/OR 按钮 | 下拉切换 AND / OR |
| 编辑条件 | 点击条件卡片 | 下方详情面板切换到该条件 |
| 条件顺序 | 拖拽条件卡片 | 调整条件排列顺序（纯展示，不影响逻辑） |

**阈值类型详细说明**：

| 类型 | 计算公式 | 示例 | 适用场景 |
|------|----------|------|----------|
| 绝对值 | `当前值 < 阈值` | 注册成功率 < 95% | 指标本身低于安全线 |
| 环比 | `(当前值 - 上周期值) / 上周期值 × 100% < 阈值` | 环比 < -20% | 突变检测（闪断/突发） |
| 同比 | `(当前值 - 昨日同时值) / 昨日同时值 × 100% < 阈值` | 同比 < -15% | 日间规律偏差检测 |

**运算符选项**：
- 数值/百分比型指标：`<` `<=` `>` `>=` `=` `!=`
- 环比/同比模式固定使用 `<`（下降超过阈值），界面自动锁定运算符

## 五、三屏联动交互流程

### 5.1 联动跳转路径

```
┌──────────┐    告警详情→查看趋势     ┌──────────┐
│          │ ──────────────────────→ │          │
│  监控页   │                         │  报表页   │
│          │ ←────────────────────── │          │
└──────────┘    异常点→查看规则       └──────────┘
      │                                    │
      │         告警详情→查看规则          │
      └──────────────→┌──────────┐←───────┘
                       │          │
                       │  规则页   │
                       │          │
                       └──────────┘
                          │
                     规则保存→刷新报表/监控
```

### 5.2 联动上下文传递

| 跳转方向 | 传递上下文 | 目标页行为 |
|----------|-----------|-----------|
| 监控→报表 | ne_id + 告警时段(start_at~end_at) | 自动定位到该网元+聚焦时段 |
| 监控→规则 | rule_id | 自动打开规则详情/编辑 |
| 报表→规则 | ne_id + ne_type | 打开该网元类型的规则列表 |
| 报表→监控 | ne_id + 时间范围 | 打开该网元的告警事件 |
| 规则→报表 | ne_type + 规则关联网元 | 刷新报表页数据 |

### 7.1 键盘快捷键

| 快捷键 | 功能 | 适用页面 |
|--------|------|----------|

### 7.2 加载状态

| 场景 | 处理方式 |
|------|----------|
| 首次加载 | 骨架屏(Skeleton) |
| 数据刷新 | 表格/图表局部loading |
| 长时查询 | 进度条 + "查询中，预计N秒" |
